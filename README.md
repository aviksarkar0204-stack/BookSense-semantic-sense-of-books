## 📖 About

**BookSense** is an AI-powered semantic book recommendation system that understands the *meaning* behind your query — not just keywords. Describe a feeling, a theme, or a type of story, and BookSense will surface books that genuinely match what you're looking for.

Unlike traditional keyword search, BookSense uses **dense vector embeddings** to capture semantic similarity between your query and book descriptions. This means a query like *"a haunting story about grief and loss"* can surface relevant books even if those exact words never appear in their descriptions.

This project was inspired by a YouTube tutorial on book recommender systems, but significantly extended with:
- **HuggingFace sentence transformers** instead of paid OpenAI embeddings (completely free)
- **Emotion/sentiment scoring** on book descriptions using a distilRoBERTa model
- **Zero-shot text classification** for Fiction/Nonfiction categorization using BART-MNLI
- A custom **aesthetic Gradio UI** with dark theme and gold accents

## 🧠 How It Works

### 1. Data Cleaning (`data-exploration.ipynb`)

- Downloads the [7K Books with Metadata](https://www.kaggle.com/datasets/dylanjcastillo/7k-books-with-metadata) dataset via KaggleHub
- Removes books missing `description`, `num_pages`, `average_rating`, or `published_year`
- Filters out descriptions shorter than 25 words (stub/garbage descriptions)
- Engineers a `tagged_description` column: `isbn13 + " " + description` for traceable vector search
- Saves cleaned data as `books_Cleaned.csv` (~5,197 books)

### 2. Text Classification (`text_classification.ipynb`)

- Uses `facebook/bart-large-mnli` (zero-shot classification) to classify each book as **Fiction** or **Nonfiction**
- Pre-maps 12 common categories (Fiction, Juvenile Fiction, Biography, etc.) manually
- Runs zero-shot classification on the remaining ~1,454 uncategorized books
- Achieves **~78% accuracy** on a 600-book validation set
- Saves result as `books_with_categories.csv`

### 3. Emotion Scoring (`sentiment-analysis.ipynb`)

- Uses `j-hartmann/emotion-english-distilroberta-base` to score 7 emotions per sentence:
  `anger`, `disgust`, `fear`, `joy`, `sadness`, `surprise`, `neutral`
- Splits each description into sentences, runs the classifier on each, and takes the **max score** per emotion across all sentences
- This captures the "peak emotional moment" of a book's description
- Saves final dataset as `books_with_emotions.csv`

### 4. Vector Search (`vector-search.ipynb` + `gradio_app.py`)

- Loads `tagged_description.txt` using LangChain's `TextLoader`
- Splits into chunks using `CharacterTextSplitter` (chunk_size=1000, separator=`\n`)
- Embeds chunks using `HuggingFaceEmbeddings(model_name="all-MiniLM-L6-v2")` — a 384-dim dense embedding model specifically trained for semantic similarity
- Stores embeddings in a **Chroma** in-memory vector database
- At query time: embeds the user's query → retrieves top-50 similar chunks → extracts ISBNs → filters by category/emotion → returns top-16 results

## ⚙️ Setup & Installation

### Prerequisites

- Python 3.10+
- Conda or virtualenv recommended
- GPU optional (CPU works fine for inference)

### 1. Clone the Repository

```bash
git clone https://github.com/aviksarkar0204-stack/BookSense-semantic-sense-of-books.git
cd BookSense-semantic-sense-of-books
```

### 2. Install Dependencies

```bash
pip install pandas numpy langchain langchain-community langchain-chroma langchain-huggingface
pip install sentence-transformers chromadb gradio
pip install transformers tf-keras
```

### 3. Generate the Dataset

Run the notebooks **in order**:

```
1. data-exploration.ipynb       → generates books_Cleaned.csv + tagged_description.txt
2. text_classification.ipynb    → generates books_with_categories.csv
3. sentiment-analysis.ipynb     → generates books_with_emotions.csv
```

> ⚠️ **Note:** `sentiment-analysis.ipynb` runs inference on ~5,197 books and may take **4–6 minutes** on a GPU or longer on CPU.

### 4. Run the App

```bash
python gradio_app.py
```

Then open `http://127.0.0.1:7860` in your browser.

## 💡 Key Design Decisions

**Why HuggingFace instead of OpenAI?**
The original tutorial used OpenAI embeddings which require a paid API key. `all-MiniLM-L6-v2` is a free, lightweight model (~80MB) specifically trained for semantic similarity tasks. It runs locally with no internet dependency after the initial download.

**Why ISBN-tagged descriptions?**
Each book description is prefixed with its ISBN13 before being chunked. Since multiple book descriptions can appear in a single chunk (chunk_size=1000), this allows the app to extract *all* ISBNs from a retrieved chunk, not just the first one — giving better recall.

**Why max emotion scores instead of average?**
A book might be primarily neutral but have one intensely fearful scene. Taking the max score per emotion captures the *peak emotional register* of the book rather than diluting it with neutral sentences.

---

## 📌 Limitations

- The Chroma vector DB is rebuilt in-memory every time the app starts (takes ~15–20 seconds on first launch). Persisting it to disk would speed this up significantly.
- Emotion scores are based on book *descriptions*, not the actual book content — so results may vary for books with generic or publisher-written blurbs.
- The dataset covers books published mostly before 2008 (limited by the Kaggle dataset).
